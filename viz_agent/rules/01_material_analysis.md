# Rule 01 — Material Analysis

> **Trigger**: `material_provided` — 교재·PDF·강의 노트·수식이 포함된 입력이 들어올 때 적용

이 규칙은 VizEdu 에이전트가 시각화를 생성하기 전, 반드시 입력된 자료를 먼저 체계적으로 분석하도록 강제하는 **전처리(Pre-processing) 게이트**이다.

## 1. Material 분석 의무

시각화 코드를 **한 줄도 작성하기 전에**, 다음 순서로 자료 분석을 완료해야 한다.

1. **PDF 읽기**: PDF 파일이 포함된 경우 반드시 `material-reader` 스킬을 활성화한다.
   - PDF는 브라우저로 열지 않고 텍스트/이미지 추출 도구로만 읽는다.
2. **핵심 개념 추출**: 분석 결과에서 시각화 가능한 개념 목록을 추출한다.
   - 물리/수학 변수 및 수식(LaTeX 표기 포함)
   - 그래프 종류 (r-z 평면, 시계열, 파라메트릭 변화 등)
   - 독립 변수 / 종속 변수 / 파라미터 구분
3. **사용자 보고 및 승인**: 추출된 개념 목록을 사용자에게 보여주고, 어떤 개념을 시각화할지 승인받는다.

## 2. 물리 변수 명세표(Symbol Spec) 작성

개념이 확정된 후, `/concept-to-viz` 워크플로우의 Step 3 이전에 다음 명세를 마크다운 테이블로 정리한다.

| 기호 | 의미 | 단위 | 역할 |
|---|---|---|---|
| `T` | 온도 | °C 또는 K | 종속변수 |
| `r` | 반경 방향 거리 | m | 독립변수 |
| `k` | 열전도도 | W/(m·K) | 파라미터 |

## 3. 금지 행동

- 자료를 읽지 않고 바로 파이썬 코드를 작성하는 것 **금지**
- 수식을 임의로 가정하거나 기억에 의존하여 구현하는 것 **금지**
- `plan_*.md` 기획서 없이 코드를 바로 실행하는 것 **금지** (→ Rule 02 참조)
