# /concept-to-viz

이 워크플로우는 학습 자료(교재 PDF, 강의 노트 등)를 입력받아, 발표용 고품질 시각화 자료를 최종 출력물로 생성하는 **VizEdu 에이전트의 핵심 파이프라인**이다.

## Trigger
사용자가 다음과 같이 요청할 때:
- "이 수식/교재 내용 시각화해줘"
- "이 개념을 PPT 영상으로 만들어줘"
- `/concept-to-viz` 슬래시 커맨드

## Steps

### Step 1: Material 분석 (material-reader 스킬)
1. 사용자가 제공한 자료(PDF, 수식, 설명)를 읽는다.
2. `material-reader` SKILL.md 지시에 따라 핵심 개념·수식·변수를 추출한다.
3. 추출 결과를 표로 정리하고 **사용자에게 보고 후 시각화할 개념을 선택받는다.**

### Step 2: 시각화 기획 (viz-planner 스킬)
1. `viz-planner` SKILL.md의 Template을 사용하여 기획서(`plan_*.md`)를 작성한다.
2. 출력 경로, 그림 크기, 파라미터, Colorbar 형식 등을 모두 기획서에 명시한다.
3. **사용자 기획서 승인 후 다음 단계로 진행**한다.

### Step 3: 스크립트 생성 (viz-code-generator 스킬)
1. `viz-code-generator` SKILL.md Template을 사용하여 파이썬 스크립트를 생성한다.
2. Rule 02~06 규칙이 모두 적용된 뼈대 코드를 완성한다.
3. **추가 가이드라인**:
   - Legend가 그래프 영역을 침범하거나 겹칠 경우, 그림의 너비(`FIG_W`)를 늘려 여백을 충분히 확보한다.
   - 온도(Temperature) 표기나 Colorbar 틱 수치는 기본적으로 소수점 한 자리(`%.1f`)까지만 표기되도록 제한한다.

### Step 4: 실행 전 검수 (viz-audit 스킬)
1. `viz-audit` SKILL.md의 A1~A6 체크리스트를 순서대로 점검한다.
2. 위반 항목이 있으면 자동 수정 후 재점검한다.

// turbo
### Step 5: 렌더링 실행
1. 검수 통과 후 스크립트를 실행한다.
```bash
timeout 180 uv run python3 PPT_slide/{ch}/viz_codes/{script_name}.py
```
2. 결과물 파일 크기를 확인하여 정상 생성 여부를 검증한다.

### Step 6: 결과 보고
사용자에게 생성된 파일 경로와 핵심 설정(수식, 파라미터, 렌더링 시간)을 보고한다.

### Step 7: 레이아웃 및 렌더링 주의사항 (Guidelines)
1. **Legend(범례) 오버랩 방지**: 렌더링 결과에서 범례와 그래프 또는 UI 텍스트가 겹칠 경우, 좌표를 강제로 억지로 조정하기보다 **그림 크기(폭 등)를 늘려서** 여유 자리를 확보하는 방향으로 플롯을 수정한다.
2. **온도 표기 포맷**: Colorbar 등 온도 축에 수치를 표기할 때는 시각적 깔끔함을 위해 소수점 첫째 자리(`%.1f`)까지만 표기하도록 코드를 반영한다.
